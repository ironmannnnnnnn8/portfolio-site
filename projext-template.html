Autonomous Line Following Robot

Date: September 2024
Duration: 2 months
Type: Robotics
Arduino C++ PID Control PCB Design Sensor Fusion

Project Overview

This project involved designing and building an autonomous robot capable of following complex paths using infrared sensors and advanced control algorithms. The robot was developed for a college robotics competition and achieved first place for its speed and accuracy.

The main challenge was to create a system that could handle sharp turns, intersections, and varying lighting conditions while maintaining high speed and stability. This required careful sensor placement, motor control tuning, and algorithm optimization.

[Robot Image / Video Demo]

Technical Implementation

Hardware Design

The robot was built using a custom chassis designed in SolidWorks and 3D printed. Key hardware components include:

  • Arduino Nano microcontroller for processing
  • 8-sensor IR array for precise line detection
  • L298N motor driver for dual DC motors
  • Custom PCB for clean wiring and reduced noise
  • 7.4V LiPo battery for power
[Circuit Diagram / PCB Design]

Control Algorithm

The robot uses a PID (Proportional-Integral-Derivative) controller to maintain its position on the line. The algorithm calculates error based on sensor readings and adjusts motor speeds accordingly.

// PID Control Implementation float error = calculateError(sensorValues); integral += error; float derivative = error - lastError; float correction = (Kp * error) + (Ki * integral) + (Kd * derivative); leftMotorSpeed = baseSpeed + correction; rightMotorSpeed = baseSpeed - correction; setMotorSpeeds(leftMotorSpeed, rightMotorSpeed); lastError = error;

Sensor Calibration

Implemented automatic calibration routine to adapt to different surface conditions and lighting. The robot calibrates by moving over the line and recording minimum and maximum sensor values, then uses these to normalize readings during operation.

Challenges & Solutions

1. Sharp Turn Handling

Initial implementation struggled with 90-degree turns. Solution: Implemented adaptive speed control that reduces speed when high error is detected, allowing the robot to navigate sharp corners without losing the line.

2. Lighting Variations

Performance degraded under different lighting conditions. Solution: Added ambient light compensation and implemented dynamic threshold adjustment based on real-time sensor readings.

3. Motor Speed Inconsistency

Noticed drift due to motor manufacturing variations. Solution: Implemented motor calibration factors and used encoder feedback for precise speed control.

Results & Performance

The final robot achieved impressive performance metrics:

  • Maximum speed: 2.5 m/s on straight paths
  • Successfully navigated complex tracks with 95% accuracy
  • Won first place in college robotics competition
  • Battery life: 45 minutes of continuous operation

Future Improvements

Lessons Learned

This project taught me the importance of systematic testing and iterative improvement. Key takeaways include:

  • The value of proper sensor calibration for consistent performance
  • How small mechanical imperfections significantly impact control algorithms
  • The importance of modular code design for easy debugging and tuning
  • Real-world testing is essential - simulations can't capture everything
